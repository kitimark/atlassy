## ADDED Requirements

### Requirement: CLI run command generates real UTC timestamp
The `execute_run_command()` function SHALL populate `RunRequest.timestamp` with the current UTC time in RFC 3339 format at invocation time. The timestamp MUST NOT be a hardcoded literal.

#### Scenario: Single run produces current timestamp
- **WHEN** the CLI `run` command is invoked
- **THEN** the `RunRequest.timestamp` field contains an RFC 3339 UTC timestamp reflecting the actual invocation time

#### Scenario: Successive runs produce distinct timestamps
- **WHEN** the CLI `run` command is invoked twice in sequence
- **THEN** the two `RunRequest.timestamp` values are different

### Requirement: Manifest default timestamp reflects parse time
The `default_manifest_timestamp()` serde default function SHALL return the current UTC time in RFC 3339 format when a manifest entry omits the `timestamp` field. The default MUST NOT be a hardcoded epoch or placeholder literal.

#### Scenario: Manifest entry without explicit timestamp
- **WHEN** a batch manifest JSON entry omits the `timestamp` field
- **THEN** the deserialized `ManifestRunEntry.timestamp` contains an RFC 3339 UTC timestamp reflecting the time the manifest was parsed

#### Scenario: Manifest entry with explicit timestamp
- **WHEN** a batch manifest JSON entry includes an explicit `timestamp` value
- **THEN** the deserialized `ManifestRunEntry.timestamp` retains the explicitly provided value unchanged

### Requirement: Timestamp format is RFC 3339
All generated timestamps SHALL conform to RFC 3339 format (a profile of ISO 8601). The output MUST be parseable by any RFC 3339 compliant parser.

#### Scenario: Generated timestamp is valid RFC 3339
- **WHEN** a timestamp is generated by the CLI
- **THEN** the value matches the RFC 3339 grammar (e.g., `2026-03-11T10:30:00.123456+00:00`)

### Requirement: Test timestamps remain deterministic
Hardcoded timestamps in test code (integration tests, contract validation tests) SHALL remain unchanged. Test determinism MUST NOT be affected by this change.

#### Scenario: Pipeline integration tests use fixed timestamp
- **WHEN** pipeline integration tests construct a `RunRequest`
- **THEN** they use a hardcoded timestamp literal for reproducibility
